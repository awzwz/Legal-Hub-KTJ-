# Размещение и обновление

## Целевая площадка

- Один on-premise сервер.
- Docker Compose.
- Два SSD в RAID1, UPS, контроль SMART и свободного места.
- Публикация только Caddy `80/443`; SSH доступен только административной подсети.
- Секреты хранятся в `.env.prod` вне Git с ограниченными правами доступа.

## Порядок обновления

1. Создать внешний backup и проверить его наличие.
2. Собрать versioned Docker images.
3. Выполнить миграции отдельным контейнером `migrate`.
4. Перезапустить сервисы.
5. Проверить `/readyz` и ключевые пользовательские сценарии.
6. При ошибке вернуть предыдущую версию образов и выполнить утверждённую процедуру отката.

## Ограничение текущего этапа

Compose-файл исторически находится в `deploy/aws/`, но пригоден как основа для одного on-premise
узла. До промышленного запуска его следует выделить в отдельный on-premise профиль вместе с
Prometheus, Grafana, Alertmanager и exporters.
