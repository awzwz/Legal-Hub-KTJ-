# LegalHUB: пакет для первичного рассмотрения ИБ

Дата подготовки: 2026-06-01.

Пакет описывает тестовый стенд и целевую конфигурацию первого промышленного запуска на одном
on-premise сервере. Тестовый стенд не должен содержать реальные персональные или служебные данные
до закрытия рисков со статусом `До пилота` в `08-known-risks.md`.

## Состав

1. `01-architecture.md` - границы системы и компоненты.
2. `02-data-flow.md` - основные потоки данных.
3. `03-network-access.md` - сетевые доступы и закрытые порты.
4. `04-role-matrix.md` - матрица ролей.
5. `05-security-controls.md` - реализованные и планируемые меры.
6. `06-backup-and-recovery.md` - резервное копирование и восстановление.
7. `07-deployment.md` - порядок размещения и обновления.
8. `08-known-risks.md` - открытые риски и условия допуска.
9. `09-test-evidence.md` - результаты проверок.

Перед передачей ИБ необходимо заполнить владельцев системы, внутренний домен, IP-адреса,
сроки хранения данных и место хранения внешней резервной копии.
